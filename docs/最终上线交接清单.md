# 最终上线交接清单

> 这份清单面向真正发版使用。  
> 本地开发与验收已经完成，但最终“项目上线”仍需要部署动作、生产环境检查和发布确认。

## 1. 已经确认通过的本地证据

发布前，以下内容已经在本地完成验证：

- 全量测试通过
- 前端生产构建通过
- 首页、教程详情、社区详情、投稿台的主要交互已收口
- 后台关键链路已补齐：总览、教程、社区、审核、用户、日志
- 权限边界已验：`admin / player`、禁用登录、后台访问控制
- Inertia 页面跳转已统一，下载动作已单独收口

## 2. 部署前检查

### 2.1 环境变量

- 确认 `.env` 中的数据库、缓存、队列、邮件配置正确
- 确认 `APP_ENV=production`
- 确认 `APP_DEBUG=false`
- 确认 `APP_URL` 与实际访问地址一致

### 2.2 依赖与构建

- `composer install --no-dev`
- `npm run build`
- 确认前端产物已生成到 `public/build`

### 2.3 数据库

- 执行所有未上线的迁移
- 确认角色迁移、登录禁用迁移、日志表迁移都已生效
- 如有演示数据或重置脚本，确保生产环境不会误触发

### 2.4 缓存

- 刷新配置缓存
- 刷新路由缓存
- 刷新视图缓存
- 确认 Wayfinder / Vite 相关产物与当前代码一致

## 3. 发布过程

### 3.1 发布前最后确认

- 再次确认没有未提交的本地脏改
- 再次确认构建与测试均已通过
- 再次确认关键页面没有 Vite manifest 缺失风险

### 3.2 发布执行

- 部署代码到生产
- 执行数据库迁移
- 重启 PHP / 队列 / 相关进程
- 刷新缓存
- 验证站点可访问

## 4. 上线后检查

### 4.1 页面检查

- 首页首屏是否正常
- 教程详情是否能进入工具闭环
- 社区详情是否可浏览、点赞、解答
- 投稿台是否可新建、编辑、提交、删除
- 后台是否可正常访问各模块

### 4.2 权限检查

- 未登录用户是否只能访问公开页
- `player` 是否不能进入后台
- `admin` 是否可进入后台
- 禁用登录用户是否被正确拦截

### 4.3 数据检查

- 新日志是否正常写入
- 用户管理操作是否有审计痕迹
- 导出功能是否可用

## 5. 回滚预案

如果上线后出现问题，优先按以下顺序处理：

1. 停止继续放量或切流
2. 回滚到上一个稳定版本
3. 恢复数据库或演示数据
4. 清理缓存并重新验证
5. 修复后再重新发布

## 6. 上线责任边界

本仓库已经完成的内容：

- 功能实现
- 交互统一
- 权限与关键链路验收
- 本地测试与构建验收

仍需真实发布环境完成的内容：

- 生产部署
- 生产数据库迁移
- 线上缓存刷新
- 线上访问验证
- 线上回滚确认

当前项目已经具备上线条件。
