# 最终上线判断表

> 这份文档只做一件事：帮助你判断“现在是否已经具备上线条件”，以及“哪些内容确认上线后再做”。

## 结论先说

当前项目已经满足“可以进入发布准备”的条件。

已经验证通过的核心证据：

- 全量测试通过
- 生产构建通过
- 权限与后台关键链路已补齐并验收
- Inertia 页面导航已经统一，浏览器直跳只保留在导出下载场景

如果要把上线标准分成两层，可以直接按下面判断：

## 1. 上线前必须确认

这些项如果没过，就先不要发。

### 1.1 权限与入口

- `admin / player` 角色判定正确
- 普通玩家不能进入后台
- 禁用登录用户不能继续访问受保护页面
- 后台页的页头、侧边栏、返回入口保持一致

### 1.2 后台关键链路

- 运营总览可正常访问
- 教程管理：列表、筛选、新增、编辑、删除可用
- 社区管理：列表、详情、置顶、删除、官方回复可用
- 投稿审核：列表、详情、通过、驳回可用
- 用户管理：角色切换、重置密码、禁用 / 恢复登录、导出可用
- 日志管理：筛选、详情、导出可用

### 1.3 前台关键链路

- 首页入口分流清晰
- 教程详情页“内容 + 工具”闭环可用
- ND 计算器可用
- 工具页可用
- 社区可浏览、可发帖、可回复
- 投稿台可创建、编辑、提交、删除

### 1.4 数据与发布

- 角色迁移已执行
- 登录禁用迁移已执行
- 日志表迁移已执行
- 生产环境构建可以成功生成前端资源
- 部署后页面没有 manifest 缺失问题

### 1.5 测试与回归

- 关键后台功能测试通过
- 权限边界测试通过
- 危险操作与失败分支覆盖
- 构建通过

## 2. 上线后继续做

这些项不阻塞当前发布，但适合排进下一轮。

- 权限体系细化
- 用户管理补完整运营闭环
- 日志管理升级为真正的审计中心
- 前台埋点、推荐与转化优化
- 运营报表和趋势图
- 更强的回滚、告警、备份与恢复能力
- UI 统一性继续精修
- 更严格的类型化与服务层收口

按现在的代码状态看：

- 可以上线：是
- 仍建议上线后继续优化：是
- 还有明显阻塞项：未发现

如果你想直接进入发布阶段，当前状态已经够用了。
