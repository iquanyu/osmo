# 教程内容 × 查阅工具融合方案

这份文档只保留教程内容和工具怎么互相嵌入。

---

## 1. 总体原则

- 教程负责引流和信任
- 工具负责留存和复访
- 工具要嵌入场景，不要另开孤岛

---

## 2. 核心架构：一纵一横

### 纵向（动态内容）

| 路由 | 作用 |
|---|---|
| `/tutorials` | 教程库，按 `category` 筛选（beginner / cinematic / night / vlog / creative） |
| `/tutorials/{tutorial}` | 教程详情 |
| `/community` | 社区讨论、避坑 UGC |
| `/contribute` | 玩家投稿生产链路 |

### 横向（静态工具）

| 路由 | 作用 |
|---|---|
| `/simulator` | 参数模拟器（ND、快门、ISO、分辨率等交互 + 规格说明） |
| `/assistant` | 智能场景 AI |
| `SpecsService` | 创作者预设、硬件规格文案 |

### 核心交汇点

教程详情页应直接嵌入 ND 计算器或参数校验组件。

## 5. 视觉与技术风格

| 建议 | 说明 |
|---|---|
| 极简暗黑风 | 默认 dark 更适合摄影类内容 |

## 7. 建议的目标信息架构

```text
/tutorials              内容流（现有 + 无限滚动）
/tutorials/{id}         视频/GIF + 时间戳步骤 + 内嵌 ND 计算器 + 参数复制卡
/tools                  工具箱首页（三个工具卡片）
/tools/nd-calculator    独立 ND 计算器（可书签）
/tools/specs            规格 / 续航 / 发热表
/tools/accessories      配件榜（后期）
/simulator              保留，偏「体验 / 演示」
/community              讨论 + 配件避坑 UGC
/assistant              可并入工具或教程内嵌，不必长期单独占顶栏
```

### 顶栏命名建议

| 类型 | 现有 | 对外感知 |
|---|---|---|
| 内容流 | 教程库 | 教程（场景 / 痛点） |
| 工具箱 | 参数模拟器 + 未来 /tools | 工具（ND 计算器 / 规格表 / 配件榜） |
| 社区 | 玩家社区 | 社区 |
| AI | 智能场景 AI | 并入工具或教程内嵌 |

## 8. 分阶段落地路线图

### 阶段 0：信息架构微调

- 统一内容 / 工具 / 社区的入口层级

### 阶段 1：核心交汇点

- 从模拟器抽出 `NdCalculator` 组件
- 嵌入 `tutorials/show`，预填 `tutorial.settings`
- 独立页 `/tools/nd-calculator` 与内嵌组件共用逻辑

### 阶段 2：教程内容模型升级

- 扩展 `steps` 为结构化（timestamp、image_url）
- 增加 `video_url` / `hero_gif`
- 详情页末尾「参数复制卡」
- 投稿 / 管理后台表单同步

### 阶段 3：查阅工具独立化

- `/tools/specs` 画质续航发热表（静态 JSON MVP）
- `/tools/accessories` 配件榜（社区 tag + curated MVP）
- 顶栏「参数模拟器」升级为「工具箱」入口

### 阶段 4：视觉与性能

- 教程站默认 dark 或摄影向视觉强化
- 工具组件纯客户端，首屏 Inertia shell + 即时交互

---

## 9. 明确不做（避免范围失控）

1. **不要把所有工具塞进模拟器页** — 已过重，再加会更臃肿
2. **不要先把 AI 助手做成核心留存** — 工具书签比 AI 更稳定
3. **不要第一期追求完整视频管线** — 先 embed + 结构化 steps
4. **配件榜不要第一天上复杂评分系统** — 社区 tag + 人工列表足够验证

---

## 10. 最小可行闭环（MVP）

若资源有限，按此顺序推进即可验证闭环：

1. 抽 ND 计算器组件 → 嵌入 `tutorials/show`
2. 扩展 `steps` 结构支持时间戳 + 配图
3. 详情页末尾加「参数复制卡」（基于现有 `settings`）
4. 顶栏增加「工具箱」入口，ND 计算器独立可书签
