# 部署指南

这份文档只保留 Osmo 上线到生产环境需要做的事。

---

## 1. 环境要求

- PHP 8.4+
- Composer 2.x
- Node.js 20+
- MySQL / PostgreSQL / SQLite
- Redis（可选）

## 2. 首次部署

```bash
git clone <repository-url> osmo
cd osmo

composer install --no-dev --optimize-autoloader
npm ci
npm run build

cp .env.example .env
php artisan key:generate
php artisan migrate --force
php artisan storage:link

php artisan config:cache
php artisan route:cache
php artisan view:cache
```

## 3. Web 服务器

站点根目录指向 `public/`，未命中的请求转发到 `index.php`。

生产环境记得配置 HTTPS。

## 4. 队列与定时任务

如果启用异步任务：

```bash
php artisan queue:work --sleep=3 --tries=3
```

定时任务：

```cron
* * * * * cd /var/www/osmo && php artisan schedule:run >> /dev/null 2>&1
```

## 5. 文件存储

封面图默认写入 `storage/app/public`，通过 `storage:link` 暴露为 `/storage/*`。

如果改用对象存储，在 `.env` 里改 `FILESYSTEM_DISK` 和对应密钥。

## 6. 发布更新

```bash
git pull
composer install --no-dev --optimize-autoloader
npm ci
npm run build
php artisan migrate --force
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

## 7. 健康检查

- `/` 能打开首页
- `/tutorials` 能正常列出教程
- `/admin` 只有管理员能进

## 8. 回滚建议

- 数据库保留迁移前快照
- 代码用 Git tag 或 release 分支回退
- 前端保留上一版构建产物
