# 后台与前台真实数据改造执行清单

这份文档只保留可直接执行的开发任务表、开发建议和验收标准。

如果你要看“怎么用这个项目”或“文档该先读哪份”，请去看 `docs/使用说明.md` 和 `docs/文档导航.md`。

## 1. 技术栈与依赖说明

后端已有：

- `laravel/framework`
- `inertiajs/inertia-laravel`
- `laravel/fortify`
- `laravel/wayfinder`
- `laravel/tinker`
- `laravel/pint`
- `larastan/larastan`
- `phpunit/phpunit`

前端已有：

- `@inertiajs/react`
- `@laravel/vite-plugin-wayfinder`
- `react`
- `react-dom`
- `lucide-react`
- `recharts`
- `sonner`
- `tailwindcss`

### 结论

- 基础任务不需要新增 composer 包。
- 角色认证、路由隔离、Policy、Form Request、资源转换、分页、缓存都能用 Laravel 原生能力完成。
- 如果后续要做更细粒度权限，再评估 `spatie/laravel-permission`。

## 3. 开发任务表

## 4. 任务拆分建议

### 最小可上线版本

如果想先出一个稳定可用版本，建议优先做这 5 项：

1. 角色与权限认证
2. 投稿审核闭环
3. 玩家投稿工作台
4. 教程和社区后台管理
5. 首页真实数据替换

这套做完，项目就从“可演示”提升到“可运营”。

### 更稳妥的版本

如果时间允许，再加上这些：

- 后台分页和查询减重
- 表单请求统一化
- Policy 收口
- 测试补齐
- 空态和异常提示优化

## 5. 代码层面的开发建议

### 后端

- 先做 Middleware 和 Policy，确认权限边界。
- 再拆 Form Request，集中管理校验规则。
- 投稿审核建议继续复用 `SubmissionService`，避免控制器里写业务逻辑。
- 后台统计和列表要分开查询，避免概览页越来越重。
- 对状态型数据，尽量保持固定枚举，不要后面继续新增临时字符串。

### 前端

- 保持当前 dashboard 风格，不再切换成默认脚手架样式。
- 菜单和入口根据角色动态显示。
- 后台页面统一空态、错误态、成功提示。
- 前台改真实数据时，先替换数据源，再微调 UI。

### 测试

- 管理员访问测试
- 玩家访问测试
- 投稿创建/编辑/提审测试
- 审核通过/驳回测试
- 社区管理测试
- 前台真实数据读取测试

## 6. 可选新增包说明

当前版本不强制需要新增 composer 包。下面是可选项说明：

- `spatie/laravel-permission`：适合未来要做更复杂的多角色、多权限配置，但当前两角色场景不是必须。
- `laravel/precognition`：适合想做更细的前置表单校验体验，但不是本次核心。
- `spatie/laravel-data`：适合以后引入更规范的数据传输层，现在也不是必须。

如果目标是尽快上线，建议先不加包，直接用 Laravel 原生能力完成。

## 8. 验收标准

- 两类角色边界清晰，互相看不到不该看的内容。
- 管理员能管理教程、审核投稿、管理社区。
- 玩家能完成投稿、编辑、提交、查看结果的完整闭环。
- 前台页面展示真实数据，不再依赖硬编码演示内容。
- 关键业务有测试覆盖。
