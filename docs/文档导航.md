3. `docs/文档分工表.md`
4. `docs/项目规范与复用指南.md`
5. `docs/页面级UI规范.md`

4. `docs/环境变量说明.md`
5. `docs/部署说明.md`

### 面向继续做产品能力扩展的同学

1. 根目录 `README.md`

## 2. 文档分层说明

当前文档可以分成 6 类：

| 类型 | 文档 | 作用 |
|---|---|---|
| 仓库总览 | `README.md` | 项目总入口 |
| 使用说明 | `使用说明.md` | 新同学和运营同学的完整上手说明 |
| 文档分工 | `文档分工表.md` | 文档职责边界与维护规则 |
| 开发规范 | `项目规范与复用指南.md` | 开发方式与复用原则 |
| UI 规范 | `页面级UI规范.md`、`后台动作与审计规范.md` | 页面层级、入口语气、按钮层级、后台动作口径 |
| 执行与发布 | `后台前台执行计划.md`、`最终上线验收清单.md`、`上线准备清单.md`、`上线后风险清单.md` | 开发任务、上线准备、验收和后置风险 |
| 产品方案 | `内容工具融合方案.md`、`产品增长与优化路线图.md` | 内容工具融合与增长方向 |
| 运维部署 | `环境变量说明.md`、`部署说明.md` | 环境和部署说明 |

## 4. 一句话总结

如果你只记住一条：

- 看项目先读根 `README.md`
- 做开发先读 `项目规范与复用指南.md`
- 排任务看 `后台前台执行计划.md`
- 做治理优化看 `项目优化总览.md`
