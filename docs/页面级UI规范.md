# 页面级 UI 规范

这份文档只保留三类规则：

- 页面怎么排
- 入口怎么说
- 按钮怎么分层

---

## 1. 当前 UI 规范

### 1.1 页面结构优先级

默认视觉层级：

1. 主标题 / 主说明
2. 主 CTA
3. 次级入口或目标分流
4. 内容列表 / 卡片 / 工具区
5. 低权重辅助说明

如果一个页面同时包含“内容”和“行动”，先保内容，再收动作。

### 1.2 首屏原则

首屏只保留一个主动作，再配一个低权重辅助入口。

### 1.3 入口卡原则

首页或总览页上的入口卡，建议采用“主入口更重、次入口更轻”的结构。

推荐分层：

- 教程 / 内容主路径：最重
- 工具 / 校验路径：次重
- 社区 / 讨论路径：再次一级
- 投稿 / 工作台路径：更轻，偏任务型入口

四张卡同屏时，拉开层级的手段只有五个：

- 标题字重
- 文案长度
- 卡片高度
- 颜色对比
- 是否标记为 prominent

### 1.4 详情页原则

详情页默认采用：

- 左侧主内容
- 右侧工具 / 下一步 / 快捷动作

右侧区域只保留下一步需要的东西，不写长说明。

### 1.5 工作台原则

像投稿台、后台列表页这类高密度页面，默认偏工作台风格：

- 状态优先
- 标题优先
- 操作后置
- 危险操作弱化

不要让操作按钮成为每行最显眼的内容。

## 2. 入口语气规范

### 2.1 总体语气

当前项目的入口语气统一倾向于：

- 进入
- 查看
- 提交
- 返回
- 校验

尽量少用这些词：

- 打开
- 去
- 搞一下
- 开始

不是不能用，只是在语义很自然时才用。

### 2.2 首页入口语气

首页入口用“目标导向”语气。

- 先看教程
- 先算参数
- 先看社区
- 先提交投稿

不建议：

- 进入某某功能
- 打开某某页面

首页的作用是分流，不是列菜单。

### 2.3 教程详情语气

教程详情页用“看完就做”的语气：

- 复制参数到工具
- 进入模拟器
- 立刻校验曝光
- 继续实践

语气要短，动作要清楚。

### 2.4 社区语气

社区页更适合“参与型”语气：

- 提交解答
- 查看讨论
- 参与经验分享

保持平稳，不要像系统命令。

### 2.5 投稿语气

投稿台用“工作流语气”：

- 编辑
- 提交审核
- 修改
- 删除
- 等待审核

让用户一直知道三件事：

1. 我现在在哪个状态
2. 我下一步该点什么
3. 这个操作是否会改变状态

## 3. 按钮层级规范

### 3.1 按钮优先级定义

按钮默认分四层：

1. 主按钮：页面最重要动作
2. 次按钮：辅助主流程
3. 低权重文字入口：跳转但不抢眼
4. 危险动作：删除、驳回、禁用、清空等

### 3.2 同屏按钮数量

同一视觉区域内，尽量控制为：

- 1 个主按钮
- 1 个次按钮
- 1 个危险动作

如果必须更多，优先折叠到：

- 更多菜单
- 详情抽屉
- 二级弹窗
- 右侧动作区

### 3.3 危险动作处理

危险动作要和主按钮拉开差距：

- 颜色偏红，但不要太花
- 可用图标，但不要和主按钮同权
- 文案尽量短
- 最好配确认弹窗

危险动作不应成为同一行里最抢眼的按钮。

### 3.4 列表行按钮顺序

列表行默认顺序建议：

1. 主动作：编辑 / 查看 / 进入详情
2. 次动作：提交 / 审核 / 复制
3. 危险动作：删除 / 禁用 / 驳回

这个顺序要保持稳定。

### 3.5 右侧动作区按钮顺序

右侧动作区推荐使用：

1. 去主流程
2. 去辅助流程
3. 去低频流程

动作区里如果放列表，不要每个按钮都做成主按钮。

---

## 4. 新页面接入时的检查清单

新增页面前先问四个问题：

1. 这个页面的主动作是什么？
2. 这个页面的次动作是什么？
3. 哪些动作应该弱化成文字入口？
4. 是否存在危险操作，需要确认和降权？

如果答不出来，通常说明页面层级还没理顺。

---

## 6. 一句话总结

新增页面时，先想清楚：

- 谁是主入口
- 谁是次入口
- 谁该弱化
- 谁是危险动作

只要这四件事一致，页面通常就不会乱。
